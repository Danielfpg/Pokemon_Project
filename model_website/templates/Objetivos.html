Objetivos

Objetivo general:

desarrollar un sistema en base sql y Python que permita gestionar castas Pokémon, mostrando la carta como tal y su precio en bolsa actual.


Objetivos específicos:

  • Diseñar un sistema de datos relacional en SQL que permita almacenar y organizar información detallada sobre cartas Pokémon, incluyendo su nombre, tipo, imagen, rareza y otros atributos relevantes.

  • Implementar una interfaz en Python que permita realizar operaciones CRUD (crear, leer, actualizar, eliminar) sobre las cartas registradas, facilitando la gestión personalizada de colecciones por parte de los usuarios.

  • Integrar visualmente las imágenes y la información de cada carta en el sistema web o aplicación, mostrando una vista atractiva y ordenada de los datos para una mejor experiencia del usuario.

  • Desarrollar un sistema manual de actualización de precios de las cartas, donde los usuarios puedan ingresar o modificar el valor estimado según su última revisión de mercado.

  • Establecer un módulo de consulta que permita filtrar las cartas por tipo y nombre, mejorando la búsqueda dentro de la colección.

  • Analizar la viabilidad de incorporar en el futuro una API externa para la actualización automática de precios, considerando posibles fuentes confiables de datos del mercado de cartas Pokémon.

  • Evaluar el impacto del sistema en la organización y valorización de colecciones personales, mediante retroalimentación básica de los usuarios o pruebas funcionales.
